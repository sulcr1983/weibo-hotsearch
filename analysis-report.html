Highlights

核心亮点

01
采集层分层设计
五种采集策略（RSS / Web直采 / 汽车垂媒 / Playwright渲染 / 微博免Cookie API）， 每个采集器独立封装，数据源通过 sources.yml 声明式配置， 新增媒体源只需编辑 YAML。
02
Observability 可观测性模块
完整的漏斗追踪（原始抓取 → 品牌命中 → 维度分类 → 入库）， 每条文章带 Trace ID 日志、DropReason 枚举、HTML 有效性断言， 失败现场快照保存。个人项目中极为罕见。
03
Playwright 反检测脚本
覆盖 8 个浏览器指纹特征隐藏（webdriver / plugins / languages / platform 等）， 模拟人类滚动行为、随机延迟。比不少商业爬虫做得更细致。
04
异步写入队列 + SQLite WAL
基于 asyncio.Queue 的写缓冲架构，避免高频采集时频繁连接数据库； SQLite WAL 模式 + NORMAL 同步，兼顾性能和数据安全。
Deep Dive

详细分析

01
架构设计 A-

系统采用四层流水线架构：采集层、处理层、存储层、推送层， 每层职责清晰，模块间通过数据契约（dict）解耦。

  • 采集层 — 5 种策略（RSS / Web / 垂媒 / Playwright / 微博API），独立封装
  • 处理层 — 品牌匹配 → 多级过滤（UGC / 金融快讯 / 观点）→ 评分 → 维度分类 → SimHash去重 → 事件聚类
  • 存储层 — 异步写入队列 + SQLite WAL，双库分离（文章库 + 微博库）
  • 推送层 — 日报/周报/月报 + 飞书卡片 + 邮件HTML + AI总结（DeepSeek API）

值得注意的设计决策：微博采用57~67 分钟随机间隔避免被限流； 数据源通过 sources.yml 声明式配置，新增媒体源只需编辑 YAML 文件。

02
代码质量 B+

代码风格统一，约 5,400 行 Python，几乎全覆盖类型注解， 异常处理周到（单个源崩溃不影响整体），日志分级清晰。

  • 类型安全 — 几乎所有函数都有类型签名，在 Python 个人项目中属于上游水平
  • 异常防御 — 每个采集器有独立 try/except，启动自检验证依赖+环境变量+数据库路径
  • 错误恢复 — reschedule_job 失败有 fallback 机制，能防止崩溃循环
  • 可优化 — 品牌词典存在两套副本（regex + dict），部分函数偏长，测试覆盖可扩展
03
工程化成熟度 A-

超出独立开发者平均水准的工程化能力。V4.1 到 V4.3 的迭代日志显示了真实的运维经验。

  • 版本管理 — Git commit 规范，版本号清晰，CHANGELOG 有实质内容
  • 配置体系 — .env + config.py + sources.yml 三层，敏感信息与代码分离
  • 部署闭环 — Docker RSSHub + deploy.sh 一键部署 + 启动自检
  • 运维细节 — Windows 兼容修复、jieba API 变更适配、8 天日志轮转、数据自动清理
